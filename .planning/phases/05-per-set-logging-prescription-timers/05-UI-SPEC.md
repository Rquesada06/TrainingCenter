# Phase 5 — UI Design Contract: Per-Set Logging, Prescription & Timers

> Visual and interaction contract for Phase 5. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This phase modifies TWO surfaces: the **client workout session screen** (`src/app/client/workout/session.tsx` + the row component) and the **trainer routine builder** (`src/components/routines/RoutineExerciseRow.tsx`). It introduces an **inline rest-timer bar**, a **work-timer Start control**, and **per-set logging rows**.
> Design system: **Obsidian Performance**, already implemented across v1.0. This spec extends it; it does not redefine it.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage in Phase 5 |
|-------|-------|-----------------|
| xs | 4px | Set-row column gaps, RPE stepper inner padding, badge inner padding |
| sm | 8px | Set-row vertical padding, gap between set rows, timer-bar control gap |
| md | 16px | Card horizontal padding, screen horizontal padding, timer-bar horizontal padding |
| lg | 24px | Exercise card vertical separation, builder section spacing |
| xl | 32px | Major builder section breaks |
| 2xl | 48px | (unused this phase) |
| 3xl | 64px | (unused this phase) |

Exceptions:
- **Set-input field height:** minimum 44pt tap target (iOS HIG). The numeric input cells in a set row render at `py-2` (8px) but the whole cell `Pressable` carries a 44pt minimum height.
- **Per-set done-check:** 28×28px visual circle (slightly larger than the v1.0 exercise checkbox at 24px so it reads as the row's primary affordance), with `hitSlop={{ top: 8, bottom: 8, left: 8, right: 8 }}` → ≥44pt target.
- **Inline rest-timer bar height:** 56px content + `safeAreaInsets.bottom` padding. It sits ABOVE the pinned Finish button area, not over it.
- **Timer-bar control buttons (Skip / +15s):** minimum 44pt tap target via `hitSlop`.
- **RPE stepper buttons (− / +):** 32×32px visual, `hitSlop` to 44pt.

## Typography

Two weights only: regular (400) and semibold (600). Numeric set values use the **mono** family (`JetBrainsMono-Regular`) so weight/reps/RPE columns align — this matches the mockup's tabular treatment.

| Role | Size | Weight | Family | Line Height | Usage in Phase 5 |
|------|------|--------|--------|-------------|-----------------|
| Display | 28px | 600 | mono | 1.1 | Active timer countdown remaining (mm:ss) in the inline bar |
| Heading | 20px | 600 | sans | 1.2 | Exercise card title (name) |
| Body | 16px | 400 | sans | 1.5 | Set-input values (weight/reps), RPE value, prescription target text |
| Label | 14px | 400 | sans | 1.4 | Column header caps (`SET / PESO (KG) / REPS / RPE / STATUS`), builder field labels, timer-bar controls, target/prescription captions |

> Column header caps render at 14px (not the mockup's 10px — 10px fails the project's minimum legible size; v1.0 never goes below 14px). Use `letterSpacing: 0.5`, `textTransform: 'uppercase'`, color `#888888` to preserve the caps-label feel.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0E0E0E` | Screen background, set-row input cell background, timer-bar background |
| Secondary (30%) | `#1A1A1A` | Exercise card surface, builder row surface, set-row container |
| Border | `#444444` / `#2A2A2A` | Card/input borders (#444444), list dividers + timer-bar top border (#2A2A2A) |
| Accent (10%) | `#00FF66` | Reserved list below |
| Warning | `#FFD600` | "Timed" exercise badge in builder + client card; timer-bar in final-10s urgency state |
| Destructive | `#EF4444` | None active this phase (no destructive actions — see Copywriting) |
| Primary text | `#FFFFFF` | Logged set values, exercise names, timer countdown |
| Muted text | `#888888` | Column headers, prescription targets, prefilled (un-edited) set values, builder labels |

Accent `#00FF66` reserved for (and ONLY for):
- Per-set **done-check** filled state (circle fill + `#0E0E0E` checkmark)
- The running **timer progress fill** (the elapsed portion of the inline bar / ring)
- **"FINALIZAR SESIÓN"** finish CTA (PrimaryButton solid — existing component, label updated per copy)
- The work-timer **Start** button when idle (PrimaryButton-styled accent pill on a timed exercise)
- An exercise card's **left-edge / title accent** once that exercise counts complete (≥1 set checked, D-08)
- A focused/edited set-input cell's **border** (`#00FF66`) to signal the field being typed into

Accent `#00FF66` NOT used for:
- Column header text or labels (use `#888888`)
- Prefilled but un-edited set values (use `#888888` to signal "carried over, confirm me")
- The numeric keypad (system keyboard)
- The "+15s" / "Skip" timer controls (outline/muted — they are secondary)
- RPE stepper chrome (use `#444444` borders)

> Logged-value color rule (the one-tap speed cue): a set value that is **prefilled and unconfirmed** renders `#888888`; once the client edits it OR checks the set done, the confirmed values render `#FFFFFF`. This makes "what still needs my attention" legible at a glance.

## Screens and State Contracts

### Surface A — Client Workout Session (`src/app/client/workout/session.tsx` + `src/components/workout/ExerciseRow.tsx`)

The v1.0 row was a single per-exercise checkbox with an expand/collapse detail grid. Phase 5 **replaces the per-exercise checkbox with per-set rows** inside the expanded card and **adds an inline timer bar** pinned above the finish CTA. The single-open expand behavior is retained.

#### A1 — Exercise Card (collapsed)

```
┌──────────────────────────────────────────┐
│  {Exercise name}                     [›]  │   ← Heading 20/600
│  {sets}×{repsMin}–{repsMax} · RPE {tgt}   │   ← Label 14/400/#888888
│  ▸ {N}/{sets} sets logged                 │   ← progress caption, accent if ≥1
└──────────────────────────────────────────┘
```
- Card: `bg-[#1A1A1A]`, `border-b border-[#2A2A2A]`, `px-4 py-3`.
- Title color: `#FFFFFF`; turns to "complete" treatment (accent left-edge bar 3px `#00FF66`, title stays `#FFFFFF`, no strikethrough) once ≥1 set is checked (D-08). Strikethrough is dropped — sets are individually logged now, so a whole-exercise strikethrough would misrepresent partial logging.
- Secondary line — **weighted** exercise: `{sets}×{repsMin}–{repsMax}` (e.g. `3×8–10`), append ` · RPE {targetRpe}` only when `targetRpe != null`.
- Secondary line — **timed** exercise: `{duration}s` with a "Timed" pill (`#FFD600` text, `rgba(255,214,0,0.2)` bg, `border #FFD600`, `rounded-full px-2 py-0.5`).
- Progress caption: `{checkedCount}/{sets} sets logged` (weighted) / `Hold {duration}s` (timed). Caption color `#00FF66` when `checkedCount ≥ 1`, else `#888888`.
- Mode tag (`gym only` / `home only`) from v1.0 is preserved unchanged when `modeTag != null`.

#### A2 — Exercise Card (expanded) — WEIGHTED exercise

```
┌──────────────────────────────────────────────┐
│  {Exercise name}                         [∧]  │
│  3×8–10 · RPE 8                                │
├──────────────────────────────────────────────┤
│  SET   PESO (KG)   REPS    RPE      STATUS    │  ← header caps 14/#888888
│  ─────────────────────────────────────────    │
│   1     [ 60 ]     [ 10 ]  [ 8 ▾]    ( ✓ )    │  ← logged → values #FFFFFF
│   2     [ 60 ]     [ 10 ]  [ — ]     ( ○ )    │  ← prefilled → values #888888
│   3     [ 60 ]     [ 10 ]  [ — ]     ( ○ )    │
├──────────────────────────────────────────────┤
│  [ Video / Image — 200px, unchanged ]         │
│  Notes: {trainer notes}                       │
└──────────────────────────────────────────────┘
```

- **Column grid:** 5 columns. Flex weights mirroring the mockup's 12-col (1/3/3/3/2): `SET` 0.9, `PESO` 2.6, `REPS` 2.0, `RPE` 2.2, `STATUS` 1.6. `gap: 8`. Header row + each set row share the same flex template so columns align.
- **Header row:** caps labels, 14px/400/#888888, `letterSpacing 0.5`, `textTransform uppercase`. `marginBottom: 8`.
- **Set row container:** `bg-[#0E0E0E]`, `rounded-lg`, `px-3 py-2`, `marginBottom: 8` (8-pt). When the set is checked done: `border border-[#00FF66]` at 1px (subtle), background unchanged.
- **SET cell:** the set number `1..sets`, mono 16/400. Color `#888888`.
- **PESO (KG) cell:** a tappable numeric field. Tap → system numeric keypad (`keyboardType="number-pad"`). Displays the value in **mono 16**. `#FFFFFF` if edited/confirmed, `#888888` if prefilled-unconfirmed. Empty → placeholder `–` in `#444444`. Focused cell: `border border-[#00FF66]`.
- **REPS cell:** identical behavior to PESO, integer keypad.
- **RPE cell:** NOT a free keypad. Tapping opens the **compact RPE stepper** (see New Components → RpeStepper). Displays `8`, `8.5`, etc. in mono 16; `–` (`#444444`) when blank. RPE is optional (may stay blank).
- **STATUS cell:** the per-set **done-check** — 28×28 circle. Unchecked: `border-2 border-[#444444]`, transparent fill. Checked: `bg-[#00FF66]`, Ionicons `checkmark` 16px `#0E0E0E`. `hitSlop` to 44pt. Checking a set **auto-starts the rest timer** (D-04, see A4).
- **Prefill behavior (D-02/D-09/D-03):** weight/reps prefill from the client's last session for this exercise, falling back to the trainer target on first session. Last-edited set's values carry DOWN to the next set within the exercise. A prefilled-but-untouched set requires **only the done-check** — no re-entry. Prefilled values are `#888888` until edited or checked, then `#FFFFFF`.
- Media (video/image) and Notes from v1.0 expanded view are retained, rendered BELOW the set table.

#### A3 — Exercise Card (expanded) — TIMED exercise (D-05/D-11)

```
┌──────────────────────────────────────────────┐
│  Plank                                   [∧]  │
│  60s · Timed                                  │
├──────────────────────────────────────────────┤
│        ⟳  Start  60s                          │  ← accent Start pill (idle)
│           — or, while running —               │
│        00:42  remaining   [Skip] [+15s]       │
├──────────────────────────────────────────────┤
│  [ Video / Image ]   Notes: {trainer notes}   │
└──────────────────────────────────────────────┘
```

- **No weight/reps/RPE rows.** A timed exercise shows the **work-timer Start control** instead of a set table (D-11 — explicit toggle drives this, never inferred).
- **Start control (idle):** an accent pill, `bg-[#00FF66]`, `#0E0E0E` label `Start {duration}s`, Ionicons `play` 16px. Full-width inside the card, `rounded-lg py-3`. Tapping starts the work-timer countdown (no auto-start — D-05).
- **Running state:** Start pill is replaced by a row: countdown `mm:ss` (mono, 20/600 inline here — the big 28px display is reserved for the rest-timer bar) + `[Skip]` + `[+15s]` controls (Label 14, `#888888`, outline). A thin progress track (`#2A2A2A` background, `#00FF66` elapsed fill, 4px tall) sits under the row.
- **Done-state:** a timed exercise counts complete when its work timer has been **started at least once and run to 0 OR skipped** → marks the exercise's single implicit "set" as completed (so D-08 / `completedExerciseIds` stays consistent). Show a `✓ Done` chip (`#00FF66`).
- Alarm sound + vibration fire once at 0 (TIMR-04, see Interaction Contracts).

#### A4 — Inline Rest-Timer Bar (D-04, TIMR-01/03/04)

Pinned above the finish CTA, full-width. Appears ONLY while a rest timer is running; the client keeps scrolling and logging other sets beneath it.

```
┌──────────────────────────────────────────────┐
│ ●━━━━━━━━━━━━━━━━━━━━━○                        │  ← progress track (4px)
│  Rest   01:18                  [Skip]  [+15s] │
└──────────────────────────────────────────────┘
       (pinned ABOVE the [FINALIZAR SESIÓN] CTA)
```

- Container: `bg-[#0E0E0E]`, `border-t border-[#2A2A2A]`, `px-4`, content height 56px + bottom safe-area padding. Sits directly above the existing pinned finish-button container (which keeps its own border + safe-area padding). When the bar is visible, the FlatList `contentContainerStyle.paddingBottom` increases by 56px so the last row is never occluded.
- **Progress track:** 4px tall, `bg-[#2A2A2A]`; elapsed fill `#00FF66` animating left→right (or remaining shrinking — executor choice; remaining-shrinks is recommended so the green visibly "drains").
- **Label:** `Rest` in Label 14/400/#888888.
- **Countdown:** `mm:ss` remaining in **Display 28/600 mono `#FFFFFF`**. In the final 10 seconds, countdown + track flip to `#FFD600` (warning) for urgency. At 0: alarm + haptic fire, bar auto-dismisses.
- **Controls:** `Skip` (dismiss timer immediately, no alarm) and `+15s` (add 15s to `endsAt`). Both Label 14, `#888888`, outline pills (`border border-[#444444] rounded-lg px-3 py-2`), ≥44pt tap target. No accent — they are secondary to the running countdown.
- **Reliability:** remaining is derived from an absolute `endsAt` (D-06), recomputed on foreground; never from accumulated ticks. `expo-keep-awake` active while the bar is visible.
- Only ONE rest timer runs at a time; checking a new set while a rest timer runs **restarts** it from the new exercise's `rest` value (last-check-wins).

#### A5 — Finish CTA (existing, label updated)

- Existing pinned `FinishButton` → `PrimaryButton` solid. **Label: "Finish Session"** (English app copy; the Spanish "FINALIZAR SESIÓN" is the mockup's locale and is NOT adopted — the app ships English strings per v1.0).
- Finish-anytime (D-07): no blocking. The existing incomplete-confirm dialog is retained but its threshold now uses **per-exercise completion derived from sets** (an exercise is complete when ≥1 set checked, D-08). Unlogged sets save `null`/`completed:false`.

│  ┌─ Timed  [ ○────  off ]                   │  ← toggle, off

│  [ 90 ]                                     │

- Row card unchanged: `bg-[#1A1A1A]`, `border border-[#444444]`, `rounded-lg p-3 mb-2`.
- **Sets** field: numeric, reuses `TextField` (`keyboardType="number-pad"`).
- **Rep range:** TWO numeric fields `repsMin` and `repsMax` with an en-dash `–` separator (16/400/#888888) between them. Labeled once as "Reps (min–max)". Validation: `repsMin ≤ repsMax`, both ≥1.
- **Target RPE:** optional numeric field (1–10, 0.5 steps allowed), reuses `TextField`. Blank allowed.

## Interaction Contracts

### Per-set logging
- Tap a weight/reps cell → system numeric keypad opens, value editable. On blur, value commits to sessionStore (live state, crash-safe; D-LOG-04 single Firestore write on finalize only).
- Tap the RPE cell → RpeStepper opens; − / + adjust by 0.5; Clear returns to blank. RPE is optional.
- Tap the STATUS done-check → toggles `completed` for that set AND auto-starts the rest timer (weighted exercise, D-04). Un-checking does NOT stop a running rest timer (the timer is decoupled once started).
- Prefill: each set seeds from last-session actuals → trainer target fallback. Within an exercise, editing set N carries those values down as the prefill for set N+1 (D-02). Prefilled-unconfirmed values render muted (`#888888`).
- An exercise counts complete when ≥1 of its sets is checked (D-08) → contributes to `completedExerciseIds`, which still drives adherence (HIST-04) and `StatusBadge`.

### Rest timer (auto, weighted)
1. Client checks a set done → rest timer starts from that exercise's `rest` seconds.
2. Inline `RestTimerBar` appears above the finish CTA; FlatList bottom padding grows by 56px. Client keeps scrolling/logging.
3. `endsAt = Date.now() + rest*1000`; remaining recomputed on a ~250ms tick AND on app-foreground from `endsAt` (never accumulated ticks — D-06).
4. `expo-keep-awake` active while the bar is visible.
5. Final 10s: countdown + track turn `#FFD600`.
6. At 0: fire `expo-audio` alarm asset + `expo-haptics` vibration ONCE (TIMR-04); bar auto-dismisses; keep-awake released.
7. `Skip` → dismiss immediately, no alarm. `+15s` → `endsAt += 15000`.
8. Checking another set while running → restart from the new exercise's `rest` (last-check-wins; one timer at a time).

### Work timer (manual, timed)
1. Client taps `Start {duration}s` on a timed exercise card → countdown begins (no auto-start, D-05).
2. Same `endsAt` / keep-awake / foreground-recompute reliability as the rest timer (D-06).
3. `Skip` / `+15s` available. At 0: alarm + haptic once; marks the timed exercise's implicit set complete → "Done" chip; contributes to `completedExerciseIds`.

### Builder timed toggle
- Toggling ON hides Reps-range / Target-RPE / Rest and shows Duration; toggling OFF reverses. The hidden fields retain their RHF values (not cleared) so an accidental toggle is reversible without data loss. Persisted as an explicit boolean on the exercise.

### Finish (D-07)
- Always tappable. If every exercise has ≥1 checked set → finish immediately. Otherwise → the incomplete-confirm dialog (copy above). On confirm: build `loggedExercises` (sets with `null` for unlogged), derive `completedExerciseIds`, single Firestore write via `stripUndefinedDeep` + `withSaveFeedback`, then existing celebration flow.
