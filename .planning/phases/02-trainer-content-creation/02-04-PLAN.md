<objective>
Build the routine builder — the second layer of the content pipeline. Routines reference exercises (built in Plan 02-02) and are themselves referenced by programs (Plan 02-05). Covers all seven ROUT requirements:

- ROUT-01: Create a named routine by selecting exercises from the trainer's library
- ROUT-02: Override sets/reps/duration/rest per exercise within the routine
- ROUT-03: Add a free-text note per exercise within a routine
- ROUT-04: Reorder exercises via drag-and-drop using `react-native-reanimated-dnd` (per D-3 CONTEXT.md)
- ROUT-05: Define a gym/home alternative exercise per row via `alternativeExerciseId` cross-reference
- ROUT-06: Edit and delete routines
- ROUT-07: List all the trainer's routines

Key library decisions per D-Libraries:
- `react-native-reanimated-dnd` Sortable (installed in 02-01) for drag-reorder per D-3
- `@gorhom/bottom-sheet` (installed in 02-01) for exercise picker and alternative-exercise picker per D-Libraries

The plan is autonomous=false because Task 3 includes a checkpoint:human-verify for the drag-and-drop UX (animation behavior must be eyeball-tested on a real device per RESEARCH.md Pitfall 1 — the Reanimated v4 + reanimated-dnd combination is new and may flicker).

Output: Full routine builder with multi-exercise selection, per-exercise overrides, notes, drag-reorder, alternative-exercise wiring, and edit/delete. Plan 02-05 (programs) can `useRoutines()` to populate the day picker.
</objective>
