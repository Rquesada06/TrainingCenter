## Spacing Scale

Declared values (must be multiples of 4). Established in Phase 1; unchanged for Phase 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, dot indicator margin |
| sm | 8px | Row separator, badge horizontal padding |
| md | 16px | Screen horizontal padding, card padding, row padding |
| lg | 24px | Section gaps, avatar margin-top, EmptyState vertical padding |
| xl | 32px | Major section breaks |
| 2xl | 48px | EmptyState top/bottom padding (paddingVertical: 48) |
| 3xl | 64px | Not used in Phase 4 |

**Exceptions:**
- Touch targets: all interactive elements must be ≥ 44dp (RN: minHeight/minWidth 44). The "Change photo" affordance on the avatar must meet 44dp minimum.
- ClientPhoto avatar on profile screens: 96dp (established in Phase 2 `[clientId].tsx`).
- ClientPhoto avatar on list items: 48dp (established in Phase 2 `ClientListItem`).
- FlatList footer ActivityIndicator: 48dp paddingVertical wrapping.

## Typography

Exactly 4 sizes, 2 weights. Do NOT introduce a 5th size. Source: Obsidian Performance system established Phase 1; confirmed in codebase audit.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | Session row date + routine name; profile field labels; exercise name in session detail |
| Label | 14px | 400 | 1.5 | Muted sub-labels, email display, section headers, empty state message body |
| Heading | 20px | 600 | 1.2 | Screen titles within cards (e.g. "Session History"), section sub-headings |
| Display | 28px | 600 | 1.2 | Screen-level page titles ("History", "Profile") |

**Notes:**
- The existing codebase uses `fontSize: 24, fontWeight: 'bold'` for screen titles in some Phase 2 screens. For Phase 4, align to Display (28px, weight 600) for new screens and match the 24px pattern on modified screens to avoid visual regression. New screens introduced in Phase 4 use 28px Display.
- Weight 400 = `fontWeight: '400'`; Weight 600 = `fontWeight: '600'`. Do not use `'bold'` — use `'600'` for consistency.
- The existing `ClientListItem` uses `fontSize: 12` for sub-labels (`NoProgramIndicator`, program name). This is a pre-existing exception in the Phase 2 code; the adherence % label added in Phase 4 must use 14px (Label) to match the established 4-size system. The 12px in Phase 2 code is a legacy inconsistency — do not extend it.

**Accent (#00FF66) reserved for:**
1. "Completed" status badge text (history rows)
2. PrimaryButton solid background — existing component
3. AdherenceBadge text at 80–100% range
4. ClientPhoto placeholder initial letter — existing component
5. `ActivityIndicator color` on loading states — existing pattern
6. EmptyState CTA button background (dark #0E0E0E text on green) — for actionable lists only
7. "Saved" confirmation text on profile save success

**Warning (#FFD600) reserved for:**
1. "Partial X/Y" status badge text (history rows) — D-05
2. Existing "No active program" dot + label in `ClientListItem` — Phase 2 pattern, unchanged

**Adherence color thresholds (HIST-04 — Claude's discretion):**
Low (< 50%): `#FFD600` (yellow — calm, not red-alarm per scope instruction)

High (80–100%): `#00FF66` (green — on track)

The threshold colors are text-only on the adherence badge — no background fill. The existing row background (#1A1A1A) provides sufficient contrast.

### 2. SessionListItem

**Layout:**
```
Pressable bg=#1A1A1A borderWidth=1 borderColor=#444444 borderRadius=8
  paddingVertical=12 paddingHorizontal=16 marginBottom=8
  flexDirection=row alignItems=center
    View flex=1
      Text date (formatted "Jun 3, 2026") — Body (16px/400) color=#FFFFFF
      Text routineName — Label (14px/400) color=#888888 marginTop=2
    StatusBadge (right-aligned)
```

**Accessibility:** `accessibilityRole="button"`, `accessibilityLabel="{routineName} on {date}, {status}"` (e.g. "Upper Body on Jun 3, 2026, Completed"). Minimum touch target: 44dp height (paddingVertical=12 + 16 body + 14 label + 2 gap = ~44dp; verify and add minHeight=44 if needed).

**Date format:** Locale-friendly short form. Use `new Date(session.date + 'T00:00:00').toLocaleDateString('en-US', { month: 'short', day: 'numeric', year: 'numeric' })` — always append `T00:00:00` to prevent UTC midnight shift. Result: "Jun 3, 2026".

### 5. AdherenceBadge + ClientListItem Modification (HIST-04)

**New component:** `src/components/clients/AdherenceBadge.tsx`

**Layout:**
```
View flexDirection=row alignItems=center marginTop=2
  Text "{N}% adherence" — Label (14px/400) color: [see thresholds]
```

**Color thresholds:**
- `null` (not started / no scheduled days yet): render nothing (return null)
- 0–49%: `#FFD600`
- 50–79%: `#FFFFFF`
- 80–100%: `#00FF66`

**ClientListItem modification:**
Add below the `programLabel` slot:
```
{adherence !== null && <AdherenceBadge adherence={adherence} />}
```

The adherence value is computed by `computeAdherence(assignment, sessions, today)` from `src/lib/adherence.ts`. Fetched via a per-client `useQuery` in `ClientListItem` (lazy, MVP-appropriate for ~5 clients).

**Loading state:** While sessions are loading for adherence, render nothing in the adherence slot (avoid layout shift). Do not show a spinner per row — the existing program label loading placeholder is sufficient visual feedback.

**Accessibility:** `accessibilityLabel="Adherence: 82 percent"` on the AdherenceBadge container.

### 6. Profile Edit Screens (PROF-01 / PROF-02)

**Files modified:**
- `src/app/client/profile.tsx`
- `src/app/trainer/profile.tsx`

**Full layout (both screens identical structure):**
```
SafeAreaView bg=#0E0E0E
  ScrollView paddingHorizontal=16 paddingBottom=40

    Text "Profile" — Display (28px/600) color=#FFFFFF marginBottom=24

    ── Avatar section ──────────────────────────────────────────────
    View alignItems=center marginBottom=24
      Pressable onPress={handleChangePhoto} accessibilityRole=button
               accessibilityLabel="Change profile photo"
        ClientPhoto photoURL={user.photoURL} name={user.name} size=96
        View  [overlay affordance — bottom-right of avatar]
          bg=#1A1A1A borderWidth=1 borderColor=#444444
          borderRadius=12 padding=6 position=absolute bottom=0 right=0
          Ionicons name="camera-outline" size=16 color=#FFFFFF
      Text "Change photo" — Label (14px/400) color=#888888 marginTop=8
    ── uploading state overlay on avatar ────────────────────────────
    When isUploading:
      View absolute inset=0 bg=rgba(0,0,0,0.6) borderRadius=48 alignItems=center justifyContent=center
        ActivityIndicator color=#00FF66 size=small

    ── Name edit card ───────────────────────────────────────────────
    View bg=#1A1A1A borderRadius=8 padding=16 marginBottom=16
      Text "EDIT PROFILE" — Label (14px/600) color=#888888 marginBottom=8
      TextField label="Name" value={nameValue} onChangeText={setNameValue}
      PrimaryButton label="Save Name" onPress={handleSaveName}
                    loading={isSavingName} disabled={!dirty || !nameValue.trim()}
      savedConfirmation → Text "Saved" Label (14px/400) color=#00FF66 marginTop=8 textAlign=center
      saveError → Text errorMessage Label (14px/400) color=#EF4444 marginTop=8 textAlign=center

    ── Sign out ─────────────────────────────────────────────────────
    View flex=1 justifyContent=flex-end paddingBottom=24
      PrimaryButton label="Sign out" variant=outline onPress={handleSignOut}
```

**"Change photo" interaction flow (D-08 / D-09 / D-10):**
1. Tap avatar or "Change photo" text → `ActionSheet` / `Alert.alert` with two options: "Take photo" and "Choose from library" (+ "Cancel").
2. On choice: call `ImagePicker.launchCameraAsync` or `ImagePicker.launchImageLibraryAsync` with `{ allowsEditing: true, aspect: [1,1], quality: 0.7, mediaTypes: 'images' }`.
3. If not cancelled: enter `isUploading=true` state → show overlay ActivityIndicator on avatar.
4. Call `uploadProfilePhoto(uid, result.assets[0].uri)` → `updateUserProfile(uid, { photoURL: downloadURL })`.
5. On success: `isUploading=false`; `ClientPhoto` re-renders with new URL via TanStack Query cache invalidation.
6. On error: `isUploading=false`; show Alert with "Could not upload photo. Please try again."

**Disabled state during upload:**
- "Save Name" button: disabled while `isUploading` is true (prevent concurrent writes).
- "Change photo" affordance: opacity=0.5, non-pressable while `isUploading` is true.
- "Sign out" button: not disabled during upload (allow escape).

**Error copy:**
- Photo upload failure: Alert title "Photo upload failed", message "Could not upload your photo. Please check your connection and try again."
- Name save failure: inline text below "Save Name" button — "Could not save name. Please try again." color=#EF4444

**Permissions (iOS):** System permission dialog is triggered by expo-image-picker automatically. App must include config plugin entries in `app.config.js` (see RESEARCH.md Pitfall 2). No custom permission UI needed.

### 7. EmptyState Component (D-11 / D-12)

**New component:** `src/components/ui/EmptyState.tsx`

**Anatomy:**
```
View alignItems=center justifyContent=center paddingVertical=48 paddingHorizontal=24
  [icon slot] marginBottom=16
  Text title — Heading (20px/600) color=#FFFFFF textAlign=center marginBottom=8
  Text message — Label (14px/400) color=#888888 textAlign=center lineHeight=21
  [optional CTA button] marginTop=24
    PrimaryButton label={ctaLabel} onPress={onCta}
```

**Props interface:**
```typescript
interface EmptyStateProps {
  icon: React.ReactNode;         // Ionicons element; caller provides
  title: string;
  message: string;
  ctaLabel?: string;             // Omit for read-only lists (D-12)
  onCta?: () => void;
}
```

**CTA button:** Use `PrimaryButton` (solid variant) for consistency. Not a custom-styled Pressable.

**Accessibility:** The `icon` slot is decorative (`accessibilityElementsHidden={true}` on the icon wrapper). Title and message are read as static text. CTA button carries `accessibilityRole="button"`.

**Per-screen instances:**

| Screen | Icon | Title | Message | CTA |

| Programs (trainer) | `calendar-outline` size=40 color=#444444 | "No programs yet" | "Build a multi-week program by assigning routines to days." | "+ New Program" |
| Clients (trainer) | `people-outline` size=40 color=#444444 | "No clients yet" | "Add your first client to get started." | "+ Add Client" |
| Session history (client) | `time-outline` size=40 color=#444444 | "No sessions yet" | "Your workout history will appear here after you complete your first session." | none |
