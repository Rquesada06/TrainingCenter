| client/trainer→Firestore (own user doc) | User writes own name/photoURL; firestore.rules enforce field allowlist |
| device→camera/photo library | OS permission boundary mediated by expo-image-picker |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-11 | Tampering | uploadProfilePhoto target path | mitigate | Path is always `users/${uid}/profile.jpg` from the authenticated uid; storage.rules (04-02) reject any path where auth.uid != userId. A user cannot upload to another user's avatar. |
| T-04-12 | Elevation of Privilege | name/photoURL write | mitigate | updateUserProfile writes only name + photoURL; firestore.rules deny any update touching role/trainerId. No privileged field is writable from the profile screen. |
| T-04-13 | Information Disclosure | photo library access | accept | iOS/Android permission strings (04-02 config plugin) gate library/camera access; user explicitly grants. No background access. |
| T-04-SC | Tampering | npm/pip/cargo installs | mitigate | No installs in this plan; expo-image-picker was installed under the 04-02 supply-chain checkpoint. |
</threat_model>
