<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client app → Firestore | The finish flow writes a session document. |
| device storage (AsyncStorage) | In-progress checks + mode persisted on-device via sessionStore. |
| embedded media | exercise `videoUrl`/`imageUrl` are trainer-authored URLs rendered by expo-video/expo-image. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Spoofing | Finish session write | mitigate | The Session record sets `clientId = uid` and `trainerId` from the assignment the client owns; `firestore.rules` sessions block enforces `clientId == request.auth.uid` server-side. |
| T-03-10 | Elevation of Privilege | Double-tap finish (duplicate write, WORK-09) | mitigate | `FinishButton` is `disabled`/`loading` while `mutation.isPending`; after write, store is cleared and Home re-queries to the done state so the finish path is gone. |
| T-03-11 | Tampering | Embedded video/image URL | accept | URLs originate from the trainer-authored snapshot (immutable, written under trainer rules in Phase 2). expo-video/expo-image render remote media; no script execution surface. Non-sensitive, trainer-trusted content. |
| T-03-12 | Information Disclosure | readOnly completed-session view | mitigate | The completed session is the client's own (read filtered by clientId; rules enforce ownership). No other client's data is reachable. |
| T-03-SC | Tampering | npm installs | accept | No package-manager installs in this plan (expo-video installed + verified in Plan 02). |
</threat_model>
