# Phase 3 — UI Design Contract: Client Workout Execution

> Visual and interaction contract for Phase 3 frontend screens. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This phase is CLIENT-side only. All screens render inside `src/app/client/`.

| Property | Value | Source |

## Spacing Scale

Declared values (all multiples of 4 — mirrors `src/constants/theme.ts` `Spacing` object):

| Token | Value | Usage in Phase 3 |
|-------|-------|-----------------|
| xs | 4px (`Spacing.one`) | Icon gaps, badge inner padding, checkbox margin |
| sm | 8px (`Spacing.two`) | Exercise row internal gap, tag pill padding, chip gap |
| md | 16px (`Spacing.three`) | Screen horizontal padding, card padding, row vertical padding |
| lg | 24px (`Spacing.four`) | Section spacing, card vertical padding |
| xl | 32px (`Spacing.five`) | Between home-state sections, finish button bottom margin |
| 2xl | 48px | Major screen-top breathing room (state card top) |
| 3xl | 64px (`Spacing.six`) | Celebration screen hero area |

Exceptions:
- Touch targets for Checkbox, toggle segments, and the mode tag: minimum 44pt hit area (iOS HIG) — achieved via `hitSlop={{ top: 8, bottom: 8, left: 8, right: 8 }}` on small elements.
- Tab bar height: `56 + safeAreaInsets.bottom` (already established in client `_layout.tsx`).
- Finish button bottom clearance: `xl` (32px) above the safe area inset to sit above the tab bar.

## Typography

All sizes use system font. Two weights only: regular (400) and semibold (600).

| Role | Size | Weight | Line Height | Usage in Phase 3 |
|------|------|--------|-------------|-----------------|
| Body | 14px | 400 | 1.5 | Exercise detail text (notes, rest label, sets/reps value), muted meta text, mode tag label |
| Label | 16px | 400 | 1.4 | Exercise row name (uncompleted), state card body copy, session meta |
| Heading | 20px | 600 | 1.2 | Routine name header on workout screen, home-state card title |
| Display | 28px | 600 | 1.1 | Celebration screen primary congratulatory line |

> Source: extrapolated from established font sizes in `sign-in.tsx` (text-sm=14, text-base=16, text-4xl≈36) and `RoutineExerciseRow.tsx` (fontSize 15/16, fontWeight '600'). Sizes rounded to the closest consistent step.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0E0E0E` | Screen backgrounds, scrollable areas |
| Secondary (30%) | `#1A1A1A` | Exercise cards, home-state cards, toggle background |
| Border | `#444444` / `#2A2A2A` | Card borders (#444), dividers (#2A2A2A) |
| Accent (10%) | `#00FF66` | Reserved list below |
| Warning / Secondary accent | `#FFD600` | Mode tag "gym only" / "home only" badge |
| Destructive | `#EF4444` | Incomplete-finish confirm button tint |
| Primary text | `#FFFFFF` | All body and heading text |
| Muted text | `#888888` | Labels, placeholders, secondary meta |

Accent `#00FF66` reserved for:
- Active toggle segment fill (gym/home toggle — the selected side)
- Checkbox checked state (border + fill or checkmark)
- "Start Workout" and "Continue Workout" CTA button (PrimaryButton solid variant)
- "Finish Workout" CTA button (solid variant)
- Active exercise row name when completed (strikethrough text + green checkmark)
- Celebration screen completion count (`X of Y done` highlighted number)
- Routine name chip on the Active Workout state card

Accent `#00FF66` NOT used for:
- Unchecked checkboxes (use `#444444` border)
- Inactive toggle segment
- Body copy or labels

### Screen 2: Workout Session (`src/app/client/workout/session.tsx`)

**Layout:** `SafeAreaView flex-1 bg-[#0E0E0E]`. Header row (fixed, not scrollable) + `FlatList` of exercise rows + sticky bottom finish button.

**Resume prompt:** On mount, after sessionStore hydration (`hasHydrated()`), if `sessionStore.isActive && sessionStore.date === today && sessionStore.clientId === uid`: show `Alert.alert("Resume workout?", "You have an unfinished workout. Resume where you left off?", [{ text: "Start over", style: "destructive", onPress: clearAndStart }, { text: "Resume", style: "default", onPress: restoreSession }])`.

#### 2A — Header Area

```
[←]  {Routine name}              [Gym] [Home]
     Week {W+1} · Day {D+1}
```
- Back button: Ionicons `arrow-back` 24px, color `#FFFFFF`, `p-2` tappable area. Tap navigates back to Home (prompts save-progress if session is started — see 2D).
- Routine name: Heading (20px/600/#FFFFFF), `numberOfLines={1}`, flex-1.
- Week/Day subtitle: Body (14px/400/#888888).
- GymHomeToggle: positioned flush to the right. See component spec in New Components section.

#### 2B — Exercise Row (collapsed state)

```
┌──────────────────────────────────────┐
│  [○ checkbox]  {Exercise name}  [›]  │
│               {N}×{reps} or {dur}s   │
└──────────────────────────────────────┘
```
- Row card: `bg-[#1A1A1A]`, `border-b border-[#2A2A2A]` (no card radius — list items are flush in a bordered container), `px-4 py-3`.

- Exercise name: 16px/400/#FFFFFF when unchecked. When checked: 16px/400/#888888 + strikethrough (`textDecorationLine: 'line-through'`).
- Secondary line: sets × reps/duration in 14px/400/#888888. Format: `{sets}×{reps}` or `{sets}×{duration}s`. If both reps and duration are non-null, prefer reps.
- Mode tag (D-10): If `modeTag !== null`, render a small pill badge immediately after the exercise name on the same line (or second line if name is long): text `gym only` or `home only` in 12px/400, bg `#FFD600` with opacity 20%, border `#FFD600`, `rounded-full px-2 py-0.5`. Text color `#FFD600`.

#### 2C — Exercise Row (expanded state)

```
┌──────────────────────────────────────┐
│  [○/● checkbox]  {Exercise name}  [∧] │
│                  {N}×{reps}          │
├──────────────────────────────────────┤
│  Sets  Reps/Duration  Rest           │  ← Detail grid
│  {N}   {reps} or {dur}s  {rest}s     │
│                                      │
│  Notes: {trainer notes}              │  ← Only shown if notes ≠ null
│                                      │
│  [Video or Image — 200px height]     │  ← Only shown if videoUrl or imageUrl ≠ null
└──────────────────────────────────────┘
```
- Expanded section uses `Animated.View entering={FadeIn.duration(200)}` (mirroring existing `collapsible.tsx` pattern).
- Detail grid: horizontal row with 3 labeled cells, each `bg-[#0E0E0E]`, `rounded-lg p-2`, `flex-1`.
  - Cell label: 12px/400/#888888 (e.g. "Sets", "Reps", "Rest")
  - Cell value: 16px/600/#FFFFFF
  - Duration cell label: "Duration" with value `{N}s`.
  - Rest cell value: `{N}s`.
- Notes: 14px/400/#888888, top padding `sm` (8px).
- Video (`expo-video` VideoView): width 100%, height 200px, `contentFit="contain"`, `nativeControls={true}`, `bg-[#000000]`. Only mounted when row is expanded (one VideoPlayer instance per row — never shared across rows, per Android constraint).
- Image (`expo-image` Image): width 100%, height 200px, `contentFit="contain"`. Shown when `imageUrl ≠ null` and `videoUrl === null`.
- If both `videoUrl` and `imageUrl` are non-null: prefer `videoUrl` (show video, not image).

#### 2D — Navigation Guard

If the client taps the system back button or the header back arrow while a session is started (sessionStore.isActive = true) and at least one exercise has been checked: show `Alert.alert("Leave workout?", "Your progress is saved. You can continue this workout later today.", [{ text: "Stay", style: "cancel" }, { text: "Leave", style: "default", onPress: navigateBack }])`. The session state in AsyncStorage is NOT cleared on back — only on finish or start-over.

#### 2E — Finish Button (always visible)

```
[Finish Workout]   ← PrimaryButton solid, full-width, bottom-safe-area clearance
```
- Pinned to screen bottom: `position absolute bottom-0 left-0 right-0 px-4 pb-[safeAreaInsets.bottom + 16]`.
- Uses `PrimaryButton` solid variant (`bg-[#00FF66]` label `#0E0E0E`).
- `disabled={mutation.isPending}` (WORK-06 race guard).
- If all exercises checked: tap directly triggers the finish mutation (no confirm).
- If 1+ exercises unchecked: `Alert.alert("Finish workout?", "You've completed {checked} of {total} exercises. Save anyway?", [{ text: "Cancel", style: "cancel" }, { text: "Finish", style: "default", onPress: finishMutation }])`.
- Loading state: `PrimaryButton loading={true}` (shows ActivityIndicator, `opacity-50`).

### Screen 4: Read-Only Completed Session View

**Reuse approach:** The session screen (`workout/session.tsx`) accepts a `readOnly` prop (or a route param `readOnly=true`). When in read-only mode:
- All checkboxes are rendered in their final `completedExerciseIds` state, visually checked or unchecked, but the `Pressable` is disabled (`pointerEvents="none"` wrapper).
- GymHomeToggle is visible but non-interactive (visual only, shows the mode the session was completed in).
- Finish button is replaced by a `PrimaryButton variant="outline"` labeled "Close" that navigates back.
- Header shows "Session Complete" instead of the routine name.
- Row expansion still works for viewing exercise detail (read-only, no interaction change needed for expansion).

This reuse avoids a dedicated summary component while keeping the completed-state visual identical to the live session. The checked/unchecked state is driven by the `completedExerciseIds` array from the Firestore session document (not the sessionStore).

## Interaction Contracts

### Expand/Collapse

- Only one exercise row expanded at a time. Opening a new row collapses the previously open one (controlled `expandedExerciseId` state in the parent FlatList component).
- Animation: `FadeIn.duration(200)` from `react-native-reanimated` on the expanded content — mirrors existing `collapsible.tsx` pattern.
- Chevron icon rotates from `chevron-down` to `chevron-up` (no animation, instant).

### Checkbox

- Tap checkbox OR tap the row (not the expand chevron area) to toggle completion.
- Expand chevron tappable area is isolated to the right 44pt.
- Immediate optimistic update in sessionStore — no loading state on individual checkboxes.

### Gym/Home Toggle

- Tap a segment to switch mode. `sessionStore.setMode()` is called immediately.
- All exercise rows re-resolve via `resolveVariant()` — the FlatList re-renders.
- Last chosen mode persisted across sessions via `AsyncStorage.setItem('laufit:lastWorkoutMode', mode)`.
- No confirmation needed for toggle — it is reversible and non-destructive.

### Finish Workout Flow

1. Client taps "Finish Workout".
2. If `completedCount === totalExercises`: skip confirm, go to step 4.
3. If `completedCount < totalExercises`: `Alert.alert` confirm (see Copywriting).
4. Button shows loading (`mutation.isPending`), disabled to prevent double-tap.
5. `createSession()` writes to Firestore.
6. `sessionStore.clearSession()` clears local state.
7. Navigate to `client/workout/celebration` (modal presentation).
8. On "Back to Home": `router.dismissAll()` + `queryClient.invalidateQueries(['todaySession'])`.
9. Home screen re-queries — `useTodaySession` now returns a session → State 1f renders.

### Date Stale-Check on App Open

- On `client/index.tsx` mount: if `sessionStore.date !== null && sessionStore.date !== localTodayString()`, call `sessionStore.clearSession()` before evaluating any resume state.
