<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| device → Firebase Auth | onAuthStateChanged reflects server-truth auth state; client cannot forge it |
| navigation layer (client-side) | Stack.Protected is a UX guard; real access control is Firestore rules (Plan 04) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | auth session | mitigate | Auth state derived only from Firebase `onAuthStateChanged` (server truth); native token rotation on sign-in. No client-settable auth bypass. |
| T-02-02 | Elevation of Privilege | role-based routing | mitigate | Role read from Firestore USERS doc on each auth event, not from URL or client input; Stack.Protected blocks deep links into the wrong role's group. Server-side enforcement is Firestore rules (Plan 04). |
| T-02-03 | Information Disclosure | cold-start render | mitigate | Root layout returns null until isLoaded; SplashScreen.preventAutoHideAsync prevents rendering protected/sign-in screens before auth resolves (no flash leaking that a user is/ isn't signed in). |
</threat_model>
